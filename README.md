# 贪婪之岛指定口袋卡册 H5

面向手机竖屏和微信分享的《HUNTER×HUNTER》贪婪之岛 No.000—099 三语互动卡册。

## 已实现

- 仿贪婪之岛卡册封面，点击进入第一页。
- 每页 10 个卡槽，共 10 页。
- 点击卡槽查看高清卡牌，卡牌详情内继续左右滑动切换。
- 卡册左右滑动或按钮翻页。
- 输入 `000—099` 快速跳转并高亮对应卡槽。
- 按编号、卡名和效果搜索；搜索覆盖中文、日文和英文数据。
- 中文、日文、英文卡面与文字切换。
- 手机竖屏、安全区、短屏幕和减少动画偏好适配。
- 缩略图与高清图分离，当前页按需加载。
- GitHub Pages 静态构建与自动部署工作流。

## 本地运行

要求 Node.js 22.13 或更高版本。

```bash
npm ci
npm run dev
```

打开 `http://localhost:3000/`。

## 验证

```bash
npm run build
npm test
```

静态站点输出到 `out/`。测试会检查三语 100 条卡牌数据、三语高清图和缩略图数量，以及主要交互是否存在。

## GitHub Pages

1. 将本目录作为一个 GitHub 仓库推送到 `main` 分支。
2. 在仓库 `Settings → Pages → Build and deployment` 中选择 `GitHub Actions`。
3. 推送后，`.github/workflows/deploy-pages.yml` 会构建并发布站点。

工作流默认按项目站点部署，即地址形如：

```text
https://<username>.github.io/<repository>/
```

如果仓库本身命名为 `<username>.github.io`，请把工作流中的 `PAGES_BASE_PATH` 改为空字符串。

## 数据与图片

- 网页使用 300 张移动端高清 WebP 和 300 张缩略图。
- 原始日文、中文、英文图片保留在上级工作区的 `Greed_Island_Cards/` 中。
- 中文可搜索文本由卡面 OCR 建立，并对异常卡名做了校正；它不是官方翻译文本数据库。
- 本项目是非官方爱好者图鉴，不与版权方或官方发行方关联。

社交分享预览图由内置 imagegen 生成，最终提示词以深靛色卡册、绿色靶心、金色剑形装饰和三语图鉴定位为核心。
